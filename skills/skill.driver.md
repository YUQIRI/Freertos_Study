# \# 嵌入式外设驱动开发规范

# \## 一、驱动开发原则

# 1\.  分层设计：驱动层（Driver）与应用层（App）分离，驱动层提供统一接口，应用层无需关注底层实现。

# 2\.  单一职责：每个驱动文件对应一个外设（如uart.c、i2c.c），不混合多个外设逻辑。

# 3\.  可复用性：驱动代码模块化，参数可配置（如UART波特率、I2C地址），便于在不同项目中复用。

# 

# \## 二、常用外设驱动规范

# \### 1. GPIO驱动

# \- 配置：明确引脚功能（输入/输出）、电平模式，封装GPIO初始化、置高、置低、翻转函数。

# \- 注意：避免频繁操作GPIO引脚，减少CPU占用；输入引脚需添加消抖处理（硬件消抖+软件消抖）。

# 

# \### 2. UART驱动

# \- 模式：优先使用DMA+空闲中断模式，实现串口数据的高效接收（避免查询式阻塞）。

# \- 配置：波特率（常用9600、115200）、数据位（8位）、停止位（1位）、校验位（无校验），统一配置标准。

# \- 功能：封装串口发送（字节、字符串、缓冲区）、接收函数，处理接收超时、数据帧错误。

# 

# \### 3. I2C驱动

# \- 配置：明确I2C地址（7位/10位）、时钟频率（100kHz标准模式、400kHz快速模式）。

# \- 处理：完善ACK/NACK响应判断、超时处理、总线死锁恢复逻辑，避免I2C总线挂死。

# \- 接口：提供I2C读写单个字节、多个字节函数，适配常用I2C从设备（如EEPROM、传感器）。

# 

# \### 4. SPI驱动

# \- 模式：主机模式为主，明确SPI模式（CPOL、CPHA），配置时钟极性、相位符合从设备要求。

# \- 控制：片选（CS）引脚严格控制，数据传输前拉低CS，传输完成后拉高CS，避免多设备冲突。

# \- 优化：高速传输场景下，使用DMA模式，减少CPU占用。

# 

# \### 5. ADC驱动

# \- 配置：明确采样通道、采样频率、采样精度（12位），开启ADC校准功能。

# \- 模式：单次采样/连续采样，优先使用DMA搬运采样数据，避免CPU频繁读取。

# \- 处理：对采样数据进行滤波（滑动平均、中值滤波），提高数据准确性。

# 

# \### 6. TIM定时器驱动

# \- 用途：定时器中断（定时任务）、PWM输出、脉冲计数。

# \- 配置：明确定时器时钟源、分频系数、自动重装值，精准控制定时时间/PWM占空比。

# \- 注意：定时中断服务函数尽量简短，避免影响实时性；PWM输出需配置引脚复用功能。

# 

# \## 三、驱动调试要求

# 1\.  每个驱动模块单独调试，确保基础功能正常（如UART能正常收发、I2C能正常读写）。

# 2\.  驱动中添加错误处理，返回明确的错误码（如驱动初始化失败、数据传输错误）。

# 3\.  禁止在驱动层调用应用层函数，保持驱动层独立性。
