# \# 嵌入式通信协议（UART/I2C/SPI）专用规范

# \## 一、通用通信原则

# 1\.  数据格式：统一数据帧格式，明确帧头、数据长度、数据内容、校验位、帧尾，避免数据解析错误。

# 2\.  校验机制：优先使用CRC校验（CRC8/CRC16），简单场景可使用奇偶校验、和校验，确保数据传输完整性。

# 3\.  超时处理：所有通信操作必须设置超时时间，避免程序阻塞在通信环节（如I2C读写超时、UART接收超时）。

# 4\.  容错处理：对接收的数据进行合法性判断（如帧头是否正确、数据长度是否匹配），丢弃异常数据，避免程序崩溃。

# 

# \## 二、UART 通信规范

# 1\.  数据帧格式（推荐）：帧头（0xAA）+ 数据长度（1字节）+ 数据内容（N字节）+ 校验位（1字节）+ 帧尾（0x55）。

# 2\.  波特率：项目内统一波特率，优先选择115200（兼顾速度与稳定性），避免频繁切换波特率。

# 3\.  数据传输：

# &#x20;   - 发送：先发送帧头，再发送数据，最后发送校验位和帧尾，确保数据顺序正确。

# &#x20;   - 接收：通过空闲中断触发数据接收完成，解析数据帧时，先判断帧头、帧尾，再校验数据，最后提取有效数据。

# 4\.  异常处理：接收超时、校验失败时，丢弃当前帧，重置接收状态，避免影响后续数据接收。

# 

# \## 三、I2C 通信规范

# 1\.  地址配置：明确I2C从设备地址（7位地址+读写位），避免地址冲突；若存在多个从设备，统一规划地址分配。

# 2\.  数据传输：

# &#x20;   - 写操作：发送从设备地址（写）→ 发送寄存器地址 → 发送数据 → 等待ACK响应。

# &#x20;   - 读操作：发送从设备地址（写）→ 发送寄存器地址 → 发送从设备地址（读）→ 接收数据 → 发送NACK结束接收。

# 3\.  总线管理：I2C总线空闲时，SDA、SCL均为高电平；传输过程中，避免总线被占用时进行其他操作，防止总线死锁。

# 4\.  常用场景：适配EEPROM、温湿度传感器（如DHT11）、OLED屏幕等I2C设备，统一驱动接口。

# 

# \## 四、SPI 通信规范

# 1\.  模式配置：根据从设备要求，统一SPI模式（CPOL=0/1，CPHA=0/1），项目内所有SPI设备使用相同模式，避免冲突。

# 2\.  数据传输：

# &#x20;   - 同步传输：SCK时钟同步，MOSI发送数据，MISO接收数据，数据传输时高位在前/低位在前统一（优先高位在前）。

# &#x20;   - 片选控制：每个SPI从设备对应独立的CS引脚，同一时间仅拉低一个CS引脚，避免多设备同时响应。

# 3\.  速率控制：根据从设备最大支持速率，配置SPI时钟频率，避免速率过高导致数据传输错误。

# 4\.  常用场景：适配SPI Flash、ADC、触摸屏等设备，封装统一的读写接口，便于复用。

# 

# \## 五、协议调试要求

# 1\.  通信过程中添加日志，打印发送/接收的数据，便于排查数据传输错误。

# 2\.  单独调试通信链路，确保无丢包、误码后，再对接应用层逻辑。

# 3\.  考虑通信干扰，对敏感通信线路（如SPI SCK、I2C SDA）添加抗干扰措施（如下拉电阻），代码中添加重试机制。
