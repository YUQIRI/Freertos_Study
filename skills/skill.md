# \# 嵌入式开发通用技能与核心规范

# \## 一、开发基础要求

# 1\.  开发语言：以标准C语言为主，禁止使用C++特性（除非项目明确要求），兼容C99标准。

# 2\.  编译器适配：兼容ARMCC、GCC编译器，代码不写编译器专属语法，避免移植问题。

# 3\.  工程通用规则：所有文件必须有头文件保护（优先使用#pragma once，兼容所有编译器）；禁止使用魔法数字，所有常量、寄存器地址统一用宏定义封装。

# 4\.  代码可读性：函数、变量、宏定义见名知意，禁止无意义命名（如a、b、temp1）；关键代码（初始化、中断、核心逻辑）必须加注释，说明功能、设计思路。

# 

# \## 二、核心开发原则

# 1\.  实时性：优先保证代码执行效率，中断服务函数仅做必要处理（如置标志位），不做阻塞操作（如延时、循环等待）。

# 2\.  稳定性：禁止内存泄漏、野指针、数组越界；全局变量尽量少用，必要时加static限制作用域，extern声明需统一管理。

# 3\.  可移植性：代码与硬件解耦，驱动层与应用层分离，便于更换主控芯片（如从STM32F103迁移到STM32F407）。

# 4\.  调试友好：关键流程添加精简日志（不影响实时性），错误码统一定义，便于定位HardFault、堆栈溢出等问题。

# 

# \## 三、通用注意事项

# 1\.  时钟配置：优先使用外部晶振，精准配置外设时钟，避免时钟频率错误导致外设工作异常。

# 2\.  看门狗：项目必须启用看门狗（独立看门狗/窗口看门狗），明确喂狗时机，防止程序跑飞。

# 3\.  低功耗：若项目有低功耗需求，需遵循Sleep/Stop/Standby模式配置规范，关闭无用外设时钟。

# 4\.  版本管理：代码添加版本注释，记录修改记录、作者、日期，便于多人协作。
