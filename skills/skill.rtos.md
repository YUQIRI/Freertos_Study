# \# RTOS 开发专用技能与规范（FreeRTOS/RT-Thread）

# \## 一、通用RTOS开发原则

# 1\.  任务管理：合理划分任务，每个任务职责单一，避免大任务（单个任务代码不超过500行）。

# 2\.  优先级配置：按任务重要性分配优先级（实时性要求高的任务优先级高，如中断处理任务、数据接收任务），避免优先级反转。

# 3\.  资源管理：共享资源（如全局变量、外设）必须通过同步/互斥机制保护（信号量、互斥锁），禁止裸操作共享资源。

# 4\.  堆栈配置：每个任务分配合理的堆栈大小（根据任务复杂度调整，避免堆栈溢出），开启堆栈溢出检测。

# 

# \## 二、FreeRTOS 专用规范

# 1\.  任务创建：使用xTaskCreate()创建任务，明确任务函数、任务名称、堆栈大小、参数、优先级，任务名称见名知意。

# 2\.  同步与互斥：

# &#x20;   - 信号量：用于任务间同步（如数据接收完成通知），使用xSemaphoreCreateBinary()创建二进制信号量。

# &#x20;   - 互斥锁：用于保护共享资源，使用xSemaphoreCreateMutex()创建，避免优先级反转（可使用xSemaphoreCreateRecursiveMutex()递归互斥锁）。

# &#x20;   - 队列：用于任务间数据传递（如UART接收数据传递给处理任务），队列长度、数据大小按需配置，避免队列溢出。

# 3\.  中断与RTOS适配：

# &#x20;   - 中断服务函数中，仅调用FreeRTOS中断安全API（如xSemaphoreGiveFromISR()），禁止调用非中断安全API。

# &#x20;   - 中断优先级：FreeRTOS配置的最低中断优先级（configLIBRARY\_LOWEST\_INTERRUPT\_PRIORITY），低于此优先级的中断可被RTOS调度。

# 4\.  延时函数：使用vTaskDelay()（相对延时）或vTaskDelayUntil()（绝对延时），禁止使用普通延时函数（如delay\_ms()），避免阻塞调度器。

# 

# \## 三、RT-Thread 专用规范

# 1\.  任务创建：使用rt\_thread\_create()创建动态任务，或rt\_thread\_init()创建静态任务，任务优先级、堆栈大小合理配置。

# 2\.  同步与互斥：

# &#x20;   - 信号量：使用rt\_sem\_create()创建，用于任务间同步、中断与任务同步。

# &#x20;   - 互斥锁：使用rt\_mutex\_create()创建，保护共享资源，支持优先级继承，避免优先级反转。

# &#x20;   - 消息队列：使用rt\_mq\_create()创建，用于任务间数据传递，支持异步通信。

# 3\.  外设适配：优先使用RT-Thread提供的设备驱动框架（如UART、I2C、SPI设备驱动），避免重复开发。

# 4\.  定时器：使用RT-Thread软件定时器（rt\_timer\_create()），替代硬件定时器，便于任务管理。

# 

# \## 四、RTOS 调试与优化

# 1\.  开启RTOS调试功能（如FreeRTOS的tracealyzer、RT-Thread的finsh控制台），监控任务运行状态、堆栈使用情况。

# 2\.  避免任务长时间阻塞：任务中若有阻塞操作（如等待信号量、队列），必须设置超时时间，避免任务永久阻塞。

# 3\.  优化任务调度：减少任务切换频率，避免高优先级任务频繁抢占低优先级任务，确保系统稳定性。

# 4\.  内存管理：使用RTOS提供的内存分配API（如pvPortMalloc()、rt\_malloc()），避免使用标准库malloc()，减少内存碎片。
