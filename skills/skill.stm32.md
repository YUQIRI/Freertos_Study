# \# STM32 开发专用技能与规范

# \## 一、主控与外设配置

# 1\.  主控适配：支持STM32全系列（重点F103、F407、F767），代码适配HAL库、LL库，不混用寄存器操作与库函数。

# 2\.  时钟配置：

# &#x20;   - 外部晶振：8MHz（F103）、16MHz（F407），倍频后对应系统时钟（72MHz、168MHz）。

# &#x20;   - 外设时钟：启用外设前必须先使能对应时钟（如GPIO时钟、UART时钟），使用完成后可按需关闭（低功耗场景）。

# 3\.  引脚配置：明确引脚复用功能（如UART\_TX/RX、SPI\_SCK/MOSI），避免引脚冲突；配置引脚电平（上拉/下拉/浮空）、输出模式（推挽/开漏）符合外设需求。

# 

# \## 二、STM32 特有规范

# 1\.  寄存器操作：仅在底层驱动优化时使用，必须封装成函数，禁止直接在应用层裸写寄存器地址（如直接操作GPIOA->ODR）。

# 2\.  HAL库使用：

# &#x20;   - 优先使用HAL库标准函数，避免重写HAL底层函数。

# &#x20;   - 中断回调函数：使用HAL库提供的回调接口（如HAL\_UART\_RxCpltCallback），不直接修改中断服务函数。

# &#x20;   - 状态判断：所有HAL函数调用后，必须判断返回值（如HAL\_OK、HAL\_ERROR），及时处理异常。

# 3\.  异常处理：

# &#x20;   - HardFault定位：添加HardFault异常处理函数，打印堆栈信息，便于排查问题。

# &#x20;   - 复位原因：启动时判断复位原因（上电复位、看门狗复位、软件复位），便于调试。

# 

# \## 三、STM32 常用配置模板

# 1\.  系统初始化：先初始化系统时钟，再初始化外设，最后初始化应用逻辑。

# 2\.  中断配置：优先级分组（推荐分组2），抢占优先级、响应优先级合理分配，避免中断嵌套冲突。

# 3\.  DMA配置：优先使用DMA传输（UART、SPI、ADC），配置DMA缓存区，避免数据丢失；开启DMA中断，处理传输完成/错误事件。
