# \# 项目专属技能与规范（基于SUMP协议的逻辑分析仪）

# \## 一、项目核心需求

# 1\.  功能定位：基于STM32主控，实现SUMP协议逻辑分析仪，支持GPIO采样、DMA搬运、定时器触发，上位机通过串口接收数据。

# 2\.  核心要求：实时性高、采样效率高、内存占用小，支持多通道采样，数据传输稳定无丢包。

# 

# \## 二、项目架构与模块划分

# 1\.  模块划分（对应工程结构）：

# &#x20;   - Driver：GPIO采样驱动、DMA驱动、TIM定时器驱动、UART驱动（SUMP协议通信）。

# &#x20;   - App：SUMP协议解析、采样控制、数据处理、上位机通信逻辑。

# &#x20;   - Utils：采样滤波、数据打包、CRC校验、延时函数（适配RTOS/裸机）。

# &#x20;   - Config：采样参数配置（采样率、通道数、触发条件）、SUMP协议参数配置。

# 2\.  流程逻辑：上位机发送SUMP协议指令 → 主控解析指令 → 配置采样参数（采样率、通道、触发方式） → GPIO采样 → DMA搬运采样数据 → 数据打包（符合SUMP协议） → UART发送给上位机。

# 

# \## 三、SUMP协议适配规范

# 1\.  协议指令：严格遵循SUMP协议标准，支持核心指令（如采样启动、采样停止、参数配置、版本查询）。

# 2\.  数据格式：采样数据按SUMP协议要求打包，帧头、数据长度、校验位符合协议规范，确保上位机正常解析。

# 3\.  通信适配：UART波特率固定为115200，数据位8位、停止位1位、无校验，与上位机保持一致。

# 4\.  指令响应：主控接收上位机指令后，及时响应（如发送确认指令、采样数据），避免上位机超时。

# 

# \## 四、采样与数据处理规范

# 1\.  GPIO采样：

# &#x20;   - 采样通道：支持至少8路GPIO采样，可配置采样通道（按需开启/关闭）。

# &#x20;   - 采样率：可配置（如100kHz、1MHz），根据上位机指令调整，确保采样实时性。

# &#x20;   - 触发方式：支持边缘触发（上升沿、下降沿）、电平触发，触发条件可通过上位机配置。

# 2\.  DMA搬运：

# &#x20;   - 采样数据通过DMA搬运到缓存区，避免CPU占用，提高采样效率。

# &#x20;   - 缓存区大小合理配置，避免数据溢出，DMA传输完成后触发中断，通知CPU处理数据。

# 3\.  数据处理：

# &#x20;   - 采样数据进行简单滤波（中值滤波），去除干扰信号，提高数据准确性。

# &#x20;   - 按SUMP协议要求打包数据，添加校验位，确保数据传输完整性。

# &#x20;   - 数据发送：采用DMA+UART发送，避免阻塞采样流程，确保采样与发送并行。

# 

# \## 五、项目专属注意事项

# 1\.  实时性保障：采样任务、数据搬运任务优先级最高，避免被其他任务阻塞；中断服务函数尽量简短，仅做必要处理。

# 2\.  内存优化：采样缓存区使用静态数组，避免动态内存分配，减少内存碎片；数据处理后及时释放无用数据，降低内存占用。

# 3\.  兼容性：适配主流SUMP协议上位机（如OpenSUMP），确保指令解析、数据传输兼容。
